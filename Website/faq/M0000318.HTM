Synopsys license problems.

Q:

I am just starting out. When I created my first HDL project, I got a red message concerning my Synopsys server initialization. It says that the feature has expired (-10,32). Then when checking the license, it says that it cannot find a valid license for Synopsys. Any ideas why? Was I supposed to get a separate license from Synopsys?

A:

Do you have a recent Xilinx license for your C: drive serial number? I had such a license for Foundation 2.1i with a "DISK_SERIAL_NUM=Cxxxxxxxx" line and I also had problems with Synopsys.

In the license file comments I had the following:

# Please note that license files which use a "C" drive serial number have
tips (except format C: :) but the license didn't work anyway.
I contacted Globetrotter and they told me their FLEXlm
doesn't support the "DISK_SERIAL_NUM=Cxxxxxxxx" format but only
"DISK_SERIAL_NUM=xxxxxxxx".  (I think this is the reason that the license 
MUST be on the C drive.  Because you cannot put the drive letter in the 
DISK_SERIAL... statement.)
Xilinx generated my license for my Ethernet card so I don't
know on if they already fixed this bug with their licenses
for disk C: or not.




Here are other problems I have encountered:


    
  
  1. The word license may be mispelled as licence (British spelling)
   The correct spelling is 'license'. 
    2. 
  
  2. You must have a license file in the following directory:
   fndtn\Synth\license.
3) We change the path in autoexec.bat for the license file to reflect 
   that it resides in the Synth\license subdirectory.
    3. 



Also, check to see if you are using a temporary license instead of the 
one you should have received if you registered at the Xilinx web site.
A temporary license contains this text:



  
#----------------------------------------------------
...
----------------------------------------------------




You get this temporary license file when you install the Foundation tools.
Go ahead and register at www.xilinx.com with your serial number and place
the license file in the location pointed to by your LM_LICENSE_FILE 
environment variable.
