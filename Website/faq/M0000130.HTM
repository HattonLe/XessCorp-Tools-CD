Mapping XSPORT arguments to XS40 Board pins

Q:

A class project was to design and implement a simple three input (A,B,C) majority tester. With such a circuit the output(F) is high for the following miniterms, 011,101,110,111. Using the Xilinx Foundation series 1.3 software, we find that when the specified pins for A=p47, B=p46, and C=p45, the high output for 011 and 101 is missing, with 110 and 111 high as expected. Switching to A=p45, B=p46, and C=p47 also results in missing high output for the same terms 011 and 101, with 110 and 111 high as expected. This is where it gets really strange. If we use A=p46, B=p45, and C=p44, then we get all of the expected high outputs. This same pattern is repeated on several different PCs with very different motherboards and CPUs, and several different XS40-005XL based boards.

A:

You are passing the wrong bit string to XSPORT when you map the inputs of the majority circuit to pins 45, 46, and 47. But the bit strings are correct if you use pins 44, 45, and 46.

For the XS40 Board, XSPORT maps the bits to the following positions:

    XSPORT b7b6b5b4b3b2b1b0
    b0    pin 44
    b1    pin 45
    b2    pin 46
    b3    pin 47
    b4    pin 48
    b5    pin 49
    b6    pin 32
    b7    pin 34

So when you issue the command XSPORT 011, then b2=0, b1=1, and b0=1. This sets pins 44 and 45 high, and pin 46 is driven low. (All the other pins are also driven low.) If your A, B, and C inputs are mapped to pins 44, 45, and 46, then A=1, B=1, and C=0 so the output will be high and the LED will light up. But if your A, B, and C inputs are mapped to pins 45, 46, and 47 then your majority circuit receives inputs of A=1, B=0, and C=0 so the LED stays dark. (Pin 47 is driven high, but this pin is not an input to the majority circuit in this case.) If you map the inputs to pins 45, 46, and 47, then you have to use the following commands to test it:

    XSPORT 0000    (C=0,B=0,A=0)
    XSPORT 0010    (C=0,B=0,A=1)
    XSPORT 0100    (C=0,B=1,A=0)
    XSPORT 0110    (C=0,B=1,A=1)
    XSPORT 1000    (C=1,B=0,A=0)
    XSPORT 1010    (C=1,B=0,A=1)
    XSPORT 1100    (C=1,B=1,A=0)
    XSPORT 1110    (C=1,B=1,A=1)
