Foundation Exit Code 80010104

Q:

When I run the the impementation step for the tripg_40 example in VHDL Design Flows Xilinx Foundation 1.5 , I get the message "Automation caused an exception, exit code 80010104" no matter if I press the Run or the Option Button in Implement Design Window.

A:

When you install the Foundation 1.5 software, you should make sure to install the XC3000 libraries in addition to the default libraries (XC4000 and XC9500). The software needs the XC3000 libraries to operate correctly.

You also need to install a patch from the Xilinx web site to correct some other problems in the F1.5 software. The patch is found at http://www.xilinx.com/support/techsup/sw_updates/sw_f15_pc.htm and is called 15_service_pack1_nt.zip. The file is said to be for windows NT, but it also works for Win95. The file is 11.4 MBytes.

Xilinx also has Service Pack 2 now available. This service pack is comprehensive and covers all the updates provided by Service Pack 1. The best way to get the latest service pack is to go to the menu at http://www.xilinx.com/support/techsup/sw_updates/index.htm

Enter in your software package (e.g. Foundation 1.5i, Alliance 1.5i etc.), then enter your operating system (e.g. PC, Solaris etc.) and hit the "generate" key.
