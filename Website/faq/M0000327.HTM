Q:

I´m using Foundation 2.1i and when i simulate this code i get erroneous results:

entity d_latch is
port (GATE, DAT

Q:

out std_logic); end d_latch; architecture rtl of d_latch is begin infer: process(GATE, DATA) begin if (GATE = '1') then Q <= DATA; end if; end process infer; end rtl;

Supposed to be a latch, but when GATE=1 Q is not transfered to DATA; and when GATE=0 Q is transfered to DATA. What´s wrong?

A:

I synthesized and simulated your code using Foundation 2.1i with SP6. I got the same result. When I inverted the polarity of the GATE signal, I got the same behavior (which is correct in this case). I checked the synthesized netlist for each design and they were both correct (i.e., the polarity of the GATE signal changed when the VHDL code changed).

I changed the code as follows so I could download it to an XS40-005XL Board (I added an input buffer and a global buffer so I could drive the GATE signal from a generic I/O pin):
