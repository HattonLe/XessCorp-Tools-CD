Connecting to clocks (CLKIOB) in VHDL

Q:

I am writing a program that allows the FPGA to read a mouse. I am trying to connect the pins in the UCF file but i get the following error:

ERROR:baste:263 - The LOC constraint "P68" (a IOB location) 
is not valid for symbol "MOUSE_CLK.PAD" (pad signal=MOUSE_CLK), 
which is being mapped to the following site types:
CLKIOB
Errors have been found in this design.  
No output files have been written.

A:

You are using the signal coming in on pin 68 as a clock to a flip-flop. The fitter software doesn't like for you to do that unless the pin is one of the primary or secondary clock input pins, which pin 68 is not. In order to get around this error, you have to instantiate a general-purpose clock buffer (BUFG) right after the input buffer (IBUF) from pin 68. Then the fitter can use the output of the BUFG to drive the clock inputs of the flip-flop.

COMPONENT ibuf PORT(i: IN STD_LOGIC; o: OUT STD_LOGIC); END COMPONENT;
COMPONENT bufg PORT(i: IN STD_LOGIC; o: OUT STD_LOGIC); END COMPONENT;
SIGNAL buf_clk0, buf_clk1: STD_LOGIC;
BEGIN
b0: ibuf PORT MAP(i=>kb_clk,o=>buf_clk0); -- buffer the clock from
b1: bufg PORT MAP(i=>buf_clk0,o=>buf_clk1); -- the keyboard
