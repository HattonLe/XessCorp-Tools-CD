Problems using XSLOAD from a ThinkPad laptop

Q:

I have an IBM ThinkPad 560E laptop running Win95, and I have XS Board Tools/Docs V1.2 installed. Per the manual, all jumpers are in their default positions, and J8 is also shorted for 3.3v operation.

When I first apply power to the board, all segments of the LED light up. I verified that some pins on the FPGA are getting power (some pins at 3.3v and others 5v).

Running XSTEST XS40-005XL echoes that the configurations are downloaded, but no change in the LEDs takes place. Downloading a simple FPGA configuration also seems to have no effect.

Per the manual's suggestions, I changed parallel port configurations from EPP to bi-directional and uni-directional modes, but got the same results.

With a voltmeter, I was able to verify that the XSPORT command sets the parallel port signals correctly (the first two are inverted...), so I think that the parallel port is working (at least for XSPORT).

A:

Yes, the LEDs should glow dimly due to the pullups of the unprogrammed FPGA.

The XSLOAD program doesn't pay attention to any feedback from the XS Board, so it wouldn't report a problem even if it existed.

I'm assuming that you ran the XSTEST program while you were in the XSTOOLS\BIN directory and that the XS Board was the only thing connected to the parallel port. I'm also assuming you had a 9VDC supply plugged into the J9 power jack.

My first thought is that it may be something odd with the parallel port of the laptop since we have had problems with them before. A good test would be to install the XSTOOLS software on a standard desktop Win95 machine and see if the same problem occurs.

You also might try removing or disabling any parallel port drivers on your laptop -- possibly they are interfering with XSLOAD.
