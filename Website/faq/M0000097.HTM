How do I access routines in the 8031 internal ROM?

Q:

The 8031 uses internal ROM between address 000...FFF if EA_ is set high (jumper J7). Which features does the ROM program have? Are there any program entry adresses which can be used by other programs?

A:

The 8031 in the socket of most XS40 Boards has no ROM so there is never any use to having EA_=1. The 8031 always fetches its instructions from the RAM. However, you can put a version of the 8051 in the socket with a program burned into ROM. Then you can set EA_=1 and use all of the RAM for data. But it is not possible for a program in RAM to use any subroutines stored in the ROM because interleaving program fetches between internal ROM and external RAM is not allowed.

Q:

I compared the UCF-file content on pp. 249/250 which connects the RAM with the schematic description. This yielded a few disagreements:

  1. For the databus D. The connections to the XCBUS are not identical: i.e the RAM-I/O bus uses XCBUS[10,80,81,41,40,39,38,35] while the uC-D bus uses XCBUS[10,80,81,35,38,39,40,41].
  2. Also the MSB of the adressbus aren't identical: the RAM uses XCBUS[51,57,50,4,83,59,58] while the uC uses for P2: XCBUS[28,60,58,50,56,51,57,59].

A:

The RAM address and data buses are scrambled to make fabrication of the XS PWB easier. It doesn't matter which RAM data pin corresponds to a bit in a byte or which address pin corresponds to a bit in the address. It is the order of the 8051 address and data pins which is important. The XSLOAD program accounts for the scrambled RAM address and data pins when it loads programs into the RAM so that the 8051 will fetch legal opcodes from the correct program addresses.

Q:

The RAMs OE_ is tied to XCBUS[61] while the uCs RD_ (which should be connected to the RAMs OE_) is tied to XCBUS[27].

A:

The 8051 doesn't get direct control of the RAM RD_ pin because the RAM can hold either instructions or data. The 8051 will lower its RD_ pin when it gets data from the RAM, but this doesn't happen when it fetches an instruction. During instruction fetches, the 8051 lowers its PSEN_ pin. The FPGA ORs the 8051 RD_ and PSEN_ outputs and uses the result to drive the RAM OE_ pin. If the 8051 RD_ pin was connected directly to the RAM OE_, then the 8051 could fetch data but not instructions.
