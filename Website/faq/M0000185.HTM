Using the XESS Boards with Linux

Q:

How do I use XESS Board with Linux?

A:

So, you've got your new XESS board, and you're itching to start programming it. Only one catch: You decided to install Linux on your machine last month. Now you need to run the Xilinx tools on the Linux box. Yes, there is a way!

Here's what you need:

  1. Xilinx Foundation 1.5 CD 
  2. VMWare beta (http://www.vmware.com/) 
  3. NT 4.0 
  4. NT service pack 3 (http://www.microsoft.com) 
  5. Xilinx Foundation 1.5 service pack 1 (ftp://ftp.xilinx.com/pub/swhelp/M1.5_updates/15_service_pack1_nt.zip) 
  6. Xilinx Foundation 1.5 ftp update #2 (ftp://ftp.xilinx.com/pub/swhelp/M1.5_updates/15_sp1_ftp2_nt.zip) 
  7. XSTOOLS for Linux ftp://ftp.microux.com/pub/pilotscope/xstools.tar.gz

Get VMware (it's a free beta), follow the directions, and set up NT *with* service pack 3.

Now install the Xilinx foundation software on your NT system. It will ask you to reboot. (The nice thing is you only need to reboot vmware.) Then you need to unzip the service pack to a temp directory and run the setup program. Finally, unzip the last update directly into the C:\fndtn\ directory.

That's what it took me to get a running system. I would expect it would work fine on NT. I would guess win95 and win98 would work also since people are using them on real machines. It seems that my setup is about as reliable as I'm used to for Windows stuffs. (I've *never* had good luck with it, although friends I know have done well with it.)

Now, my next project is to build a makefile to run all the xilinx tools from dosemu or wine when I get really bored. (If anyone has any success running foundation under WINE, let me know.)
