How can I simulate the GNOME microcomputer as it executes an instruction?

Q:

How can I simulate the GNOME microcomputer as it executes an instruction?

A:

1) Load the Xilinx Student Edition 1.3 onto a PC.

2) Create a NEW project called GNOME_40.

3) Copy the GNOME1.ABL file from the GNOM1_40 project included in the XSE.

4) Open the GNOME1.ABL file in the HDL editor and synthesize it as a CHIP (not a MACRO).

5) Open the simulator.

6) Add these signals to the waveform viewer window: addr14-addr0, din7_PIN-din0_PIN, clock_PIN, and reset_PIN.

7) Attach the B0 bit of the binary counter in the stimulator window to the clock_PIN input.

8) Force the din7_PIN..din0_PIN bus to 01010000 (this is the opcode for an ADD operation between the accumulator and data register 0).

9) Attach the reset_PIN signal to the 'R' key of the keyboard. Press the R key until the reset_PIN is forced to a logic 1 (reset enabled).

10) Do a single simulation step. This causes 5 clock cycles to occur. After this, the GNOME should be properly reset. Then press the R key to bring the reset_PIN to logic 0 (reset disabled).

11) Do a few more simulation steps. The GNOME will execute several ADD R0 instructions. When GNOME is fetching instructions the address bus progresses as follows: 0x0000, 0x0001, 0x0002, ... When the GNOME is executing the ADD R0 instruction, it reads the contents of R0 from the RAM at address 0x7FF0.
