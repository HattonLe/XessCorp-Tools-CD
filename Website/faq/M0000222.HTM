IPXS work-arounds for XS40 V1.4 Boards

Q:

How do I get IPXS working with the XS40 V1.4 Board?

A:

I´ve looked to the manual of the XS40 v1.4 and I´ve detected some inconsistencies with IPXS v2.0:

  1. The default oscillator frequency is 50 MHz. Memdrvs of IPXS are designed to work up to 12MHz. For higher frecuencies, proper operation is not warranted.
  2. XS40+ Boards have 128 KByte RAMs instead of 32 KByte RAMs.
  3. Bits C1 and C3 of the control register of parallel port are attached to different pins of the FPGA. These pins are used by the IPXS programm to control the memory drivers.
    • bit C3 is attached to p71 in version 1.4 (p15 in version 1.2)
    • bit C1 is attached to p73 in version 1.4 (p16 in version 1.2)
    A possible solution is to modify the ucf file:
    • Original not.ucf file: 
          # Control pins of Parallel port
    NET RST LOC=P15;
    NET HST LOC=P17;
    NET OP  LOC=P16;
    • Modified not.ucf file: 
          # Control pins of Parallel port
    NET RST LOC=P73;  # modified
    NET HST LOC=P17;
    NET OP  LOC=P71;# modified

BAD NEWS! P73 is a special pin of the FPGA (configuration clock CCLK) and it can´t be used as general purpose input pin. "NET RST LOC=P73" is forbidden in a .UCF file.

SOLUTION: It´s not very easy, I have to modify the IPXS code and the general interface with the memory drivers. It will takes me some weeks.

ALTERNATIVE SOLUTION:

  1. Program the oscillator fequency to 12MHz.
  2. Hold the upper two bits of the memory addres bus at logic 0. (only for XS40+)
  3. Modify the ucf file (only the indicated line) and re-compile the project:
        NET RST  LOC=P16; # not modified
    NET HST LOC=P17;
    NET OP  LOC=P71; # modified
  4. Attach the RST pin (p16 of the FPGA) to the C1 bit of parallel port (J1-14 connector). This can be made with a wire between the J3-16 pin and the J4 jumper. (J3 is the externall connector where the FPGA pins are available, J4 is a jumper next to the chip memory, see XS40 board v1.4 shcematic in the manual).
  5. This not a very good solution but is the best I can offer at the moment while I modify the IPXS code.
