Some parallel port issues that may make XS boards fail GXSTEST

Q:

I have an XS95 board with the Xilinx xc95-108 and a 8051 winbond microcontroller. I have installed the xtoolset (gxsport gxstest...) as well as the port driver port95nt on winnt 4.0 Sp6. But when ever I try to perform a test, gxstest will abort without any error message and without performing the test It is expected to do. When selecting another board type (e.g xs 40-005XL) in the gxstest interface, the program will run normally and report that the test failed (this is what I expected since I choose the wrong board type). I have the same problem on 3 different machines with the same configuration. But on 2 other machines everything works quite well. Is there any logical reason why my boards behave in this way? I have checked the power supply as well as the download cable and everything seems to be ok on this side.

A:

I will mention two, new things to check which can cause problems with the parallel port:

  1. Make sure the parallel port is not in bidirectional mode. The downloading program can accidently switch the port into input mode and this can cause problems.
  2. Remove any dongles or anything else on the parallel port other than the XS95 Board.
