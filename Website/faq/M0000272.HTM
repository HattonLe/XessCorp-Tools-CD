Compilers for the 8031 on the XS Boards

Q:

How do I test the 8031 microcontroller on the xs40 board? How would I use a C-compiler to assemble code? There's a free C compiler offered by Dunfield Development. Could that be used?

A:

I might have some useful information re C compilers for the 8031.

Over the last week, I tried 3 demo c compilers linked to the Intel 8051 developers site.

  1. The tasking 8051 environment (www.tasking.com). This compiler seems to have lots of tricks in it for optimizing the code, but either I'm stupid or the compiler is flaky (probably I'm stupid) but whenever I tried to do anything using a subroutine, the program didn't work! I think it's something about how parameters are passed into functions. This was the result after a LOT of reading of the compiler manual and messing about with different memory models.
  2. The IAR C compiler (www.iar.com). This compiler looked ok, but the demo apparently cannot make a hex output. (It seems to be specialized for use with the CSpy ROM monitor, which is some sort of system for hardware debugging your 8051 application using extra chunks of code on the 8051 that output debug information on the 8051's serial port). Like I say, I couldn't get hex output so couldn't test the code. The demo documentation comes in the form of PDF files that have no hyperlinks or table of contents so it is a bit hard to get going.
  3. The Archimedes development environment (www.keil.com/home.htm). This one produces code that seems to run, and has a nice environment. In particular, the special C constructs for refering to absolute memory locations ("_at_") are not documented in the getting started manual and I had to dig around their discussion forum. The demo does not appear to supply the full C documentation. Also, the code appears to be larger than for the tasking compiler, but maybe this is because function calling reserves the registers for other stuff, so this is not as efficient. (But it works without a lot of fiddling around, at least so far).

For all these compilers, you need to choose a memory model and data model. this is somewhat cryptic to someone coming from a UNIX environment, but if you fiddle around enough you should be able to get things to work.

It REALLY helps to have a logic analyzer hooked up to the 8031 to see what memory accesses its doing. In fact, this is the probably the most straightforward way to debug code that uses the Xilinx and the 8031, because in the simulators it is not a straightforward affair to simulate the Xilinx, if, for example, you are using the Xilinx as a memory mapped peripheral.

All these development systems cost $1k-2k commercial price for the full version, and the demos are either time limited (Tasking, 1 month) or code limited (Archimedes, 1kB).
