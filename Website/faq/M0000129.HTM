How do I use the special-purpose pins of the FPGA?

Q:

How can I use all 8 bits of the parallel port going from the PC to the XS40 v1.2 board? The problem is with the top two bits: PC_D6 and PC_D7 which are connected to Xilinx pins P32 and P34 on the XS40. When I try to constrain those signals to those pins in the .ucf file and run the Foundation tools I get the following errors from the mapping phase:

Section 1 - Errors
------------------
ERROR:baste:262 - Bad format for LOC constraint P32 on IPAD symbol
"H1/$I159" (pad signal=H1/PC_D6).  No such site for this device.  This
may also indicate that a non-constrainable site (such as a VCC, GND,
mode, configuration, or other special-purpose pin) has been used as a
site name.

ERROR:baste:262 - Bad format for LOC constraint P34 on IPAD symbol
"H1/$I160" (pad signal=H1/PC_D7).  No such site for this device.  This
may also indicate that a non-constrainable site (such as a VCC, GND,
mode, configuration, or other special-purpose pin) has been used as a
site name.

Now, it's true that these are mode pins on the XC4010 part, but they should be usable as I/O once configuration has completed. There's a comment in the .ucf file I found that says "Must use special purpose pins for accessing PC_D6 and PC_D7" but I can't interpert what that means. Is there any way to use these pins or are the upper two bits of the parallel port just not useable?

A:

Chapter 4 of "The Practical Xilinx Designer Lab Book" has a discussion on how to do this for an adder circuit. You can embed your design in a schematic and use the special symbols "MD0" and "MD2" to gain access to the upper two bits of the bus.

In VHDL, you need to instantiate modules for the mode pins. Here is a piece of VHDL that instantiates components to gain access to inputs through the M0 and M2 mode pins:

...
COMPONENT MD0 port(I: out std_logic); END COMPONENT;
COMPONENT MD2 port(I: out std_logic); END COMPONENT;						
BEGIN
u0: MD0 PORT MAP(I=>tmp1);
u1: MD2 PORT MAP(I=>tmp2);
...

Below is a VHDL example that shows how to use the mode and JTAG pins as well as the BUFG clock buffer. Generally, you have to instantiate a special component for each type of special-purpose pin. The component must then drive or be driven by an IBUF or OBUF component, respectively. The output from the IBUF can be used as an internal logic signal that carries the value from the special-purpose input pin. Any results can be driven into the input of the OBUF that then drives the special-purpose output pin.

library IEEE;
use IEEE.std_logic_1164.all;

entity special is
port ( d : out std_logic);
end special;

architecture special_arch of special is
-- special-purpose JTAG pins
component TDI  port (I : out STD_LOGIC);  end component;
component TCK port (I : out STD_LOGIC); end component;
component TMS port (I : out STD_LOGIC); end component;
component TDO port (O : in STD_LOGIC); end component;
-- special-purpose mode pins
component MD0 port (I: out STD_LOGIC); end component;
component MD2 port (I: out STD_LOGIC); end component;
component MD1 port (O: in STD_LOGIC); end component;
-- input, output, and clock buffer components
component IBUF port (I : in STD_LOGIC; O : out STD_LOGIC); end component;
component OBUF port (I : in STD_LOGIC; O : out STD_LOGIC); end component;
component BUFG port (I : in STD_LOGIC; O : out STD_LOGIC); end component;
-- intermediate signals
signal s: STD_LOGIC_VECTOR(13 downto 0);
signal j,m,c,clk: STD_LOGIC;
begin
    -- get the inputs from the JTAG TDI, TMS, and TCK pins
    u0: TDI port map(I=>s(0));
    u1: IBUF port map(I=>s(0),O=>s(1));
    u2: TMS port map(I=>s(2));
    u3: IBUF port map(I=>s(2),O=>s(3));
    u4: TCK port map(I=>s(4));
    u5: IBUF port map(I=>s(4),O=>s(5));
    -- get the inputs from the M0 and M2 mode pins
    u6: MD0 port map(I=>s(6));
    u7: IBUF port map(I=>s(6),O=>s(7));
    u8: MD2 port map(I=>s(8));
    u9: IBUF port map(I=>s(8),O=>s(9));
    -- do some logic operations on the JTAG and mode inputs
    j <= s(1) and s(3) and s(5); -- AND TDI, TMS, TCK
    m <= s(7) or s(9); -- OR MD0 and MD2
    -- output the AND result on the TDO output
    u10: OBUF port map(I=>j,O=>s(10));
    u11: TDO port map(O=>s(10));
    -- output the OR result on the MD1 output
    u12: OBUF port map(I=>m,O=>s(11));
    u13: MD1 port map(O=>s(11));

    -- get a clock input
    u14: IBUF port map(I=>c, O=>s(12));
    -- pass it through a low-skew buffer
    u15: BUFG port map(I=>s(12),O=>clk);
    -- use it to toggle a flip-flop on the rising clock edge
    process(clk)
    begin
        s(13) <= not(s(13));
    end process;
    -- output the flip-flop
    d <= s(13);

end special_arch;
