Displaying images using IPXS

Q:

I've been working with the XS-40-005XL v.1.3. board, trying to display images on the VGA monitor. I'm using IPXS software V1.0 to convert a BMP file to a hex file for the SRAM.

A:

You can get the last version of IPXS at: http://http://www.dtic.ua.es/dtic.local/asignaturas/DCAC/IPXS.htm. There is a documentation file to guide you through the IPXS issues.

Q:

At the beginning I couldn´t display anything logical, but now I am displaying some images. I have seen some issues to the IPXS software and I'd like to learn a little bit more about IPXS. First of all, the colors are not working properly. When I sent something black, IPXS converted the image to a hex file with information that correponds to red. When I sent a red screen, IPXS sent information with white. I thought solving the problem could be as easy as changing the VHDL color coding, but it wasn't so. There has to be something else with IPXS that I don't understand.

A:

IPXS only works with 256 gray level images (8bits/pixel), so when IPXS opens a BMP file it takes only one byte per pixel (the first byte in the BMP file). I recommend you transform your image to a gray image.

Q:

On the other hand, IPXS is suppossed to be able to program the FPGA with the bit file and the image I want to display, but it does not work. So I had to program the FPGA by using GSXLOAD. But I can read the image stored in the SRAM with the read option of IPXS (when I loaded the memdrvr design on the FPGA).

A:

IPXS uses the original xsload application to configure the FPGA, so xsload.exe (DOS version) has to be installed. Additionally the PATH and XSTOOLS_BIN_DIR environment variables have to be updated. The .BIT files provided with IPXS (v2.0) were implemented for XC4010 chip. If you have a 4005 FPGA, you need to re-implement the example projects.
